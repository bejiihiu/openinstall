# Let's be honest

If you've read this far, you probably use Linux too. And I don't want to sell you a project, convince you of something, or start another argument about which package manager is better. I just want to talk like a normal person. Because this whole thing has been pissing me off for a long time.

Not Linux. I love Linux. What pisses me off is how we got so used to this ecosystem that we stopped noticing how fucking weird it is for someone seeing it for the first time.

We already know on autopilot: Arch means AUR or pacman. Ubuntu means apt or .deb. Fedora means dnf or rpm. We don't even think about it anymore. It's a reflex. But you know what happens to someone who was on Windows yesterday, or installed Linux for the first time today? They open Discord's website and instead of one button, they get a goddamn exam.

Flatpak? AppImage? RPM? DEB? Or maybe from the repository? Or maybe from AUR? What's the difference? Why does one guide say one thing and another guide says something completely different? And this isn't some complex kernel tuning or driver bullshit. The person just wants to install one fucking app.

---

A while ago, a friend of mine installed CachyOS with GNOME. I'd been telling him to try Linux for a long time because the system is genuinely great. When he installed it, I was happy. He was happy too. Everything looked amazing. Beautiful GNOME, smooth animations, fast performance — the first impression was genuinely strong.

You know how long it lasted?

Until "How do I install Discord?"

And here's the thing. I caught myself sitting there, thinking. Not because I don't know Linux. I've been using it for years. But even after all this time, I have to stop and think: which install method is the right one for this case? Which package is fresher? Where are there fewer problems? What should I recommend to this specific person?

And right then I realized something.

If *I* sometimes have to pause and think — what the hell must someone feel who installed the system an hour ago? They don't understand a single thing that's happening. And the problem isn't them. The problem is that we somehow convinced ourselves this whole process is normal.

---

Ask any experienced Linux user how to install something. They'll probably answer immediately. But ask ten experienced Linux users and you'll get ten different answers.

One will say use the repository. Another — Flatpak. A third will say Flatpak is evil and you should build from source. A fourth will send you to AUR. A fifth will drop a command that stopped working a year and a half ago.

We got so used to living in this chaos that we started calling it a "Linux feature." And I don't want to accept that. Because it's not a feature. It's just shitty user experience.

---

I don't want to change Linux. Seriously. I like how it works. I like package managers. I like the whole repository idea. I like that everything is open. I like the terminal. I don't want to turn Linux into a second Windows.

But I do want to stop pretending that the first contact with this system looks normal today. Because it doesn't. It looks fucking broken. And honestly, I'm tired of watching someone who just wants to install a browser, a code editor, or Discord have to first study the internal plumbing of the entire ecosystem.

That's not right.

---

That's probably why I started this project. Not because I think I know how to build the perfect Linux. And definitely not because I want to force a new standard on everyone. I just caught myself thinking one day:

*Why the hell is there still no single "Install" button that does everything?*

Why do we keep pushing this work onto the user when a program can easily do it? Why should someone need to know what package manager they have when it can be detected automatically? Why should they have to pick a package format when it can be detected automatically? Why do we still treat this like it's the user's job?

Maybe nobody will like this idea. Maybe someone will say it's not the Linux way. Maybe someone will close the repo after the first two paragraphs. And you know what? That's fine.

But if because of this project, one person installs Linux, clicks one button, installs their first app, and doesn't even think about whether apt, pacman, or dnf did the work under the hood — then none of this was wasted.

That's the experience I always wanted.

Not a simpler Linux.

A simpler path *to* it.
